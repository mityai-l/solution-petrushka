# Задание 3: Архитектура PUSH-уведомлений

## Общая схема работы
[Мобильное приложение] → получает push-токен → отправляет на бэкенд
↓
[Сервис устройств] → хранит user_id → device_token
↓
[Бизнес-события] → Корзина (товар лежит >1 часа), Заказы (отмена), Маркетинг
↓
[Очередь RabbitMQ/Kafka] → буферизация событий
↓
[PUSH-планировщик] → решает кому отправлять, формирует текст
↓
[PUSH-адаптер] → вызывает FCM (Android) или APNs (iOS)
↓
[Устройство пользователя] → отображение уведомления

## Детальное описание компонентов

### 1. Мобильное приложение
- При запуске запрашивает разрешение на показ уведомлений
- Получает уникальный токен от FCM (Android) или APNs (iOS)
- Отправляет токен на бэкенд через API

### 2. Сервис устройств (микросервис)
- Хранит в БД: `user_id`, `device_token`, `platform`, `is_active`, `allow_marketing_push`
- При смене устройства старый токен деактивируется

### 3. Источники событий
- **Сервис корзины**: раз в 15 минут сканирует корзины с `last_updated > 1 час` → событие `cart.abandoned`
- **Сервис заказов**: при отмене заказа → событие `order.cancelled`
- **Маркетинговый сервис**: по расписанию → событие `marketing.promo`

### 4. Очередь сообщений (RabbitMQ или Kafka)
- Нужна для асинхронности и буферизации
- При 100 000 событий в секунду очередь не даст упасть push-сервису
- Топики: `cart.abandoned`, `order.cancelled`, `marketing.promo`

### 5. PUSH-планировщик
- Подписан на все топики очереди
- Проверяет настройки пользователя (`allow_marketing_push` для рекламы)
- Формирует текст уведомления (с плейсхолдерами для имени, номера заказа и т.д.)

### 6. PUSH-адаптер
- Вызывает внешние API:
  - Firebase Cloud Messaging (для Android)
  - Apple Push Notification Service (для iOS)
- Обрабатывает ошибки (невалидный токен → деактивирует)

## Почему микросервисная архитектура?

| Плюс | Объяснение |
|------|------------|
| Независимое масштабирование | Если push-сервис перегружен, корзина и заказы продолжают работать |
| Отказоустойчивость | Падение одного сервиса не останавливает другие |
| Легкость замены | Можно сменить FCM на другой провайдер, не трогая остальной код |
